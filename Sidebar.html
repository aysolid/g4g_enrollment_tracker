QuestionPro webhook setup

Use a form-specific URL so the script knows which raw tab should receive the webhook. Configure each webhook to send a POST request with the full response payload as JSON or form data.

  • Prescreening form webhook → URL ending in ?form=prescreening → response should land in Prescreening_Raw.
  • For cohort-specific prescreeners, add query values such as &cohort_id=COHORT-01&cohort_name=Cohort%201&site=UGA&program_term=Summer%202026. These values flow into clean, master, follow-up, ready, and dashboard views.
  • Consent form webhook → URL ending in ?form=consent → response should land in Consent_Raw.
  • If QuestionPro asks for content type, choose application/json.
  • After saving each webhook, submit a test response and click Refresh tracker.
